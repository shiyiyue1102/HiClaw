# Changelog (Unreleased)

Record image-affecting changes to `manager/`, `worker/`, `openclaw-base/` here before the next release.

---

- feat(hiclaw-controller): Nacos package auth is selected per-URI via the `authType` query parameter (`?authType=nacos|sts-hiclaw|none`) on `nacos://` worker/manager `package` fields, not `HICLAW_NACOS_AUTH_TYPE` (removed). `PackageResolver` always receives `CredClient` from `HICLAW_CREDENTIAL_PROVIDER_URL` when set so `authType=sts-hiclaw` works without a separate env switch.

- feat(hiclaw-controller): honor a new `spec.env` (`map[string]string`) field on Worker, Manager, Team.leader, and Team.workers[] CRs — user-declared variables are injected into the target container alongside the system-managed ones, with system-produced keys winning any collision (ignored user keys are logged at INFO level with the offending key list). Wired at `createMemberContainer` (covers standalone Workers and both Team roles via the existing `WorkerSpec` projection) and `createManagerContainer`; the `WorkerEnvBuilder` stays pure. CRDs add an `env` property block with `additionalProperties.type=string` and `propertyNames.pattern=^[a-zA-Z_][a-zA-Z0-9_]*$` so non-POSIX keys are rejected at admission. `hiclaw` CLI / REST API surface is intentionally deferred — users apply env via `kubectl apply -f` directly for now.

- feat(api)!: switch `Worker`/`Manager`/`Team` `spec.mcpServers` from `[]string` to `[]{name,url,transport}` (**breaking**). The old shape carried only a server name and relied on the controller to resolve both the endpoint URL and consumer authorization, which was ineffective in two different ways: in local Higress mode the authorization was owned by `manager/agent/skills/mcp-server-management` and the CRD field was silently ignored, and in cloud AI-Gateway mode `gateway.AIGatewayClient.AuthorizeMCPServers` was a no-op stub so the CRD list had no effect at all. The new structured item declares everything mcporter needs: `name` keys the entry in `mcporter-servers.json` (tool calls use `<name>.<tool>`), `url` is the full gateway endpoint (e.g. `https://apig.example.com/mcp-servers/github/mcp`), and `transport` is `http` (Streamable HTTP, default) or `sse`. CRD YAMLs in `hiclaw-controller/config/crd/` and `helm/hiclaw/crds/` for workers/managers/teams enforce `required: [name, url]` and the transport enum. `hiclaw-controller/internal/agentconfig/mcporter.go` is rewritten to translate the new slice directly into `mcporter-servers.json` while injecting a fixed `Authorization: Bearer <gatewayKey>` header (reusing the same consumer key the worker already uses for LLM access). `hiclaw-controller/internal/migration/registry_migration.go::extractMCPServers` now parses the legacy `mcporter-servers.json` (both `{mcpServers: {...}}` and flat-map shapes) to recover `url` and `transport` per server so v1.0.9 agents upgrade cleanly. The `--mcp-servers` flag is removed from `hiclaw create/apply/update worker`; MCP servers must now be declared in YAML manifests (see `docs/declarative-resource-management.md`).

- refactor(controller): remove MCP authorization from the gateway clients and `service.Provisioner`. `gateway.Client` loses `AuthorizeMCPServers` / `DeauthorizeMCPServers`; `HigressClient` drops `modifyMCPServers` / `listMCPServers` and related private helpers; `AIGatewayClient` drops the matching no-op stubs. `service.Provisioner` loses `ReconcileMCPAuth`, `WorkerProvisionRequest.McpServers`, `WorkerProvisionResult.AuthorizedMCPs`, `WorkerDeprovisionRequest.McpServers`, `ManagerProvisionRequest.McpServers`, `ManagerProvisionResult.AuthorizedMCPs`, and the `mcpServers` parameter on `DeprovisionManager`. `WorkerDeployRequest`/`ManagerDeployRequest` replace `AuthorizedMCPs []string` with `McpServers []v1beta1.MCPServer` and the reconcilers pass `m.Spec.McpServers` straight through. This cleanly splits responsibility: the controller only translates client-side mcporter config, while gateway-side MCP authorization is handled out-of-band (by `manager/agent/skills/mcp-server-management` for local Higress, or by the cloud operator for AI Gateway). Integration test `TestManagerUpdate_MCPServersChange_TriggersReauth` is replaced by `TestManagerUpdate_MCPServersChange_RewritesMcporterJSON` (asserts the `ManagerDeployRequest.McpServers` payload, not a gateway reauth call), and a symmetric Worker test is added.

- feat(migration): `extractMCPServers` now recovers `url` and `transport` from the legacy `mcporter-servers.json` when migrating v1.0.9 agents to v1beta1 CRs. Both shapes are accepted (`{"mcpServers": {...}}` wrapped and bare flat map), and entries missing an explicit transport default to `http`. Without this change the breaking CRD schema would have rejected migrated specs because `url` is required; with it, existing deployments upgrade in place with full endpoint information preserved. New unit tests in `hiclaw-controller/internal/migration/registry_migration_test.go` lock in both shapes.
